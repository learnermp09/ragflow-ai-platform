# 1. Overview

The RAGFlow AI Platform frontend is built using **Streamlit**. It provides a simple web interface for interacting with the Retrieval-Augmented Generation (RAG) system.

Users can enter questions in natural language and receive answers generated by the backend. The frontend communicates with the FastAPI application through REST APIs and focuses only on user interaction, leaving all business logic to the backend.

---

# 2. Frontend Architecture

The frontend follows a lightweight architecture.

```
              User
                │
                ▼
        Streamlit Interface
                │
         HTTP POST Request
                │
                ▼
        FastAPI Backend
                │
                ▼
         JSON Response
                │
                ▼
      Display Generated Answer
```

The frontend does not perform any document processing or AI inference. Its responsibility is to collect user input, communicate with the backend, and present the response.

# 6. Communication with Backend

The frontend communicates with the backend using the following REST endpoint.

### Endpoint

```
POST /api/v1/ask
```

Example request:

```json
{
    "question": "What is the American Community Survey?"
}
```

Example response:

```json
{
    "answer": "The American Community Survey (ACS) is..."
}
```

The frontend uses Python's **requests** library to send HTTP requests and process JSON responses.

# 7. Error Handling

Basic error handling has been implemented for common scenarios.

Current checks include:

- Empty question validation
- Backend connection errors
- Request timeout handling
- HTTP error responses
- Unexpected exceptions

These messages are displayed directly in the Streamlit interface to provide feedback to the user.
